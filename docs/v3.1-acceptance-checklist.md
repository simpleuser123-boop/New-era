## 发布前命令顺序

按以下顺序执行。没有运行的命令不得写成通过。

```powershell
npm run lint
npm run build
npm run smoke
npm run smoke:agent
npm run smoke:risk-benchmark
npm run smoke:risk-feedback
```

如果要验收已有服务而不是让 smoke 脚本临时启动服务，先确认目标服务使用的 SQLite 路径，再显式设置：

```powershell
$env:SMOKE_BASE_URL="http://127.0.0.1:3000"
npm run smoke
npm run smoke:agent
```

真实本地库验收必须显式选择，禁止重置、删除或重建 `data/new-era.db`：

```powershell
$env:SMOKE_USE_REAL_DB="1"
npm run smoke
```

## Local-first 与安全边界

| 边界 | 状态 | 发布前检查点 | 本轮证据 |
|---|---|---|---|
| 本地 SQLite 数据优先 | 本轮已验证 | 业务数据默认来自本地 SQLite；smoke 使用隔离库，真实库验收必须显式选择。 | `npm run smoke`、`npm run smoke:agent`、浏览器隔离库。 |
| 不自动投递 | 本轮已验证 | 文案和 Agent smoke 不得宣称已自动投递；投递类动作只能进入待确认或本地 Tracker。 | `npm run smoke:agent` 文案边界扫描；`/analytics` 文案明确不自动投递。 |
| 不自动联系 HR | 本轮已验证 | 不出现“已联系 HR / 自动发邮件 / 自动站内信”等承诺。 | `npm run smoke:agent` 文案边界扫描。 |
| 不接真实招聘平台授权 | 本轮已验证 | 不声称接入招聘平台账号、同步平台状态或读取平台私有数据。 | `DataSourceBoundary` 文案、`/assistant` 和 `/analytics` 浏览器检查。 |
| 不做招聘网站自动爬取 | 本轮已验证 | 允许用户手动输入 JD 或只读公开搜索摘要；不做平台爬取管道。 | v3.1 范围边界、Agent copy boundary scan。 |
| Query 工具只读 | 本轮已验证 | Query tools 只能读本地上下文、聚合数据或公开网页摘要。 | `npm run smoke:agent` 覆盖 10 个 Query tools。 |
| Action 工具只创建待确认 proposal | 本轮已验证 | 确认前不得写业务表；确认 / 忽略必须由用户显式触发。 | `npm run smoke:agent` 中 Action proposal 检查：未确认不改 `applications`，确认后才按 proposal 写入，忽略不写 `application_events`。 |
| 反馈不自动改规则 / 评分 / 投递状态 | 本轮已验证 | 风险反馈只能写 feedback 记录，不能改报告、应用或规则。 | `npm run smoke:risk-feedback`。 |

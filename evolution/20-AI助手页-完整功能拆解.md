## 功能拆解

### 1. 页面定位

AI 助手是统一自然语言入口，用于查询和组织本地求职数据：
- “我这周应该优先跟进哪些岗位？”
- “哪些报告显示假 AI 岗风险？”
- “帮我准备这份报告的面试问题。”
- “我的简历对 Agent 产品岗位缺什么证据？”

### 2. 子 Agent 路由

路由由 `src/lib/agent/registry` 管理：

| Agent | 负责场景 |
|------|----------|
| tracker-agent | 投递管理、跟进、状态复盘 |
| evaluate-agent | JD 评估、报告、风险 |
| resume-agent | 简历、匹配、优化、ATS |
| interview-agent | 面试准备、回答、STAR |
| career-agent | Career DNA、目标和策略 |
| general-agent | 兜底问答 |

### 3. SSE Agent Loop

`POST /api/agent/run` 返回事件流：
- `phase`
- `agent_selected`
- `tool_call`
- `tool_result`
- `action_proposed`
- `message`
- `fallback`

模型不可用或缺少 API key 时，必须明确 fallback，不伪装成模型成功。

### 4. Query Tools

Query Tools 只读：
- 搜索报告。
- 获取报告详情。
- 读取投递记录。
- 读取 Analytics 概览。
- 构建面试准备上下文。

Query 工具不能写业务表。

### 5. Action Proposals

Action Tools 只创建待确认动作：
- 创建跟进事件建议。
- 创建面试会话建议。
- 提议 Career DNA patch。

确认前不得直接写入 `applications`、`interview_sessions` 或 `settings`。用户确认后再由对应确认 API 执行动作。

### 6. 会话与记忆

会话：
- 保存标题、消息 JSON、memory digest。
- 支持读取、更新和删除。

记忆：
- 保存工作摘要等可控记忆项。
- 支持软删除。
- 页面应让用户知道哪些内容被记住。

### 7. `/assistant-preview`

`/assistant-preview` 是预览 / 对照页面，不应作为主能力文档入口。长期用户路径以 `/assistant` 为准。

## 页面状态

| 状态 | 展示 |
|------|------|
| 初始态 | 快捷问题、上下文面板、空对话 |
| 发送中 | SSE 流式事件和阶段反馈 |
| 工具调用中 | 显示 tool_call / tool_result 摘要 |
| 动作待确认 | Action proposal 卡片 |
| 会话为空 | 引导创建新会话 |
| 模型 fallback | 明确显示 fallback 原因 |
| API 失败 | 错误提示 + 保留用户输入 |

## API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/agent/run` | POST | SSE Agent loop |
| `/api/agent/actions` | GET/POST | 读取 / 创建待确认动作 |
| `/api/agent/actions/[id]/confirm` | POST | 确认执行动作 |
| `/api/agent/actions/[id]/dismiss` | POST | 忽略动作 |
| `/api/agent/sessions` | GET/POST | 会话列表 / 新建 |
| `/api/agent/sessions/[id]` | GET/PATCH/DELETE | 会话详情 / 更新 / 删除 |
| `/api/agent/memory` | GET | 记忆列表 |
| `/api/agent/memory/[id]` | GET/PATCH/DELETE | 记忆详情 / 更新 / 删除 |
| `/api/assistant` | POST | 旧兼容层 |

---

## 验收标准

- `/assistant` 可打开并发送问题。
- 用户问题能选择合适子 Agent，并显示引用或上下文来源。
- Query 工具只读，不修改业务表。
- Action proposal 未确认前不得改变 Tracker、Career DNA 或面试会话。
- 无模型、无 API key 或模型超时时，页面显示 fallback 行为。
- 会话和记忆的新增、读取、更新、删除均受用户可见控制。

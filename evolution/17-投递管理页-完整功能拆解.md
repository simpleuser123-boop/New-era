# 17 — 投递管理页完整功能拆解

> 页面: `src/app/applications/page.tsx` | 组件: `ApplicationManagementPage / ApplicationWorkspace` | API: `/api/applications/*` | 数据: SQLite `applications` / `application_events`

## 当前实现补充（2026-06-11）

投递管理是本地 Tracker，不连接招聘平台，不代替用户投递。记录可以从报告页加入，也可以通过 API 创建；所有状态、详情和时间线均保存在本地 SQLite。

## 功能拆解

### 1. 页面定位

投递管理承接报告页的“加入投递清单”动作，用于管理用户自己决定推进的岗位。

```text
评估报告
  └─ 加入投递清单 -> /applications

首页
  └─ 投递漏斗 / 最近跟进 -> /applications
```

页面必须明确：这是本地记录，不是已替用户完成平台投递。

### 2. 投递记录列表

列表每条记录展示：

| 字段 | 说明 |
|------|------|
| 公司 / 岗位 | 从报告或用户输入生成 |
| 当前状态 | 待投递、已投递、面试中、offer、已拒绝、已放弃 |
| 优先级 | 高 / 中 / 低 |
| 渠道 | Boss、猎聘、内推、官网等用户自填 |
| 风险等级 | 来自关联报告或手动记录 |
| 下一次跟进 | 用于逾期提醒 |

### 3. 搜索与筛选

- 关键词匹配公司、岗位和渠道。
- 状态、优先级、渠道筛选可叠加。
- 无结果时展示空状态和重置筛选按钮。

### 4. 列表 / 看板视图

- 列表适合快速扫描和状态更新。
- 看板按状态分列，适合复盘推进阶段。
- 当前不引入拖拽库，避免复杂状态同步。

### 5. 手动状态流转

状态流转调用：

```text
PATCH /api/applications/[id]
```

保存成功后：
- 更新列表记录。
- 写入或刷新时间线事件。
- 页面展示成功反馈。

非法状态由 API 拒绝。

### 6. 跟进详情

详情面板支持编辑：
- 来源 URL
- 渠道
- 优先级
- 下一次跟进时间
- 投递 / 面试 / offer / 拒绝 / 放弃日期
- 标签
- 备注

### 7. 时间线事件

时间线来源：
- 状态变化自动事件。
- 用户手动新增事件。

手动事件调用：

```text
POST /api/applications/[id]/events
```

### 8. 首页与 Analytics 复用

`/api/applications/stats` 和 `getAnalyticsOverview()` 会复用 Tracker 数据生成：
- 首页投递漏斗。
- Analytics 漏斗。
- 逾期跟进和活跃推进统计。

## 页面状态

| 状态 | 展示 |
|------|------|
| 加载中 | 投递清单骨架屏 |
| 空记录 | 引导去评估岗位 |
| 有数据 | 筛选器 + 统计 + 列表 / 看板 + 详情 |
| 无匹配 | 空状态 + 重置筛选 |
| 保存中 | 对应按钮禁用，显示保存提示 |
| 保存失败 | 保留草稿并展示错误 |

## API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/applications` | GET | 查询投递记录 |
| `/api/applications` | POST | 创建投递记录 |
| `/api/applications/[id]` | GET | 读取单条记录 |
| `/api/applications/[id]` | PATCH | 更新状态和详情 |
| `/api/applications/[id]/events` | GET | 读取时间线 |
| `/api/applications/[id]/events` | POST | 新增时间线事件 |
| `/api/applications/stats` | GET | 读取投递统计 |

---

## 验收标准

- `/applications` 可从侧边栏进入，导航高亮正确。
- 报告加入投递清单后，刷新 `/applications` 仍可看到记录。
- 状态、详情和时间线保存后刷新不丢失。
- 搜索和筛选可叠加，无结果状态可重置。
- 文案不得暗示已经自动投递或接入招聘平台。

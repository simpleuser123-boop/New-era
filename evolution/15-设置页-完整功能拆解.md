# 15 — 设置页完整功能拆解

> 页面: `src/app/settings/page.tsx` | 组件: `SettingsPage / SettingsWorkspace` | 数据: SQLite `settings` + auth tables

---

## 当前实现补充（2026-06-11）

- 设置页已经读取 / 保存 SQLite `settings`，不再是纯前端状态。
- 账号安全区接入 `/api/auth/me`、`/api/auth/password`、`/api/auth/logout`。
- 数据治理区接入 `/api/export` 和 `/api/data/clear`，支持导出 JSON、按类别清除本地数据和确认文本。
- 设置保存会过滤敏感 key，导出不包含环境变量密钥、token 或密码明文。
- 清除本地数据必须由用户选择类别并输入确认文本，不做隐藏自动清理。

## 功能拆解

### 1. 页面结构

```text
设置页
  ├─ Header: 标题、说明、统计卡片
  ├─ 主内容
  │   ├─ 个人信息
  │   ├─ 专业背景摘要
  │   └─ 通知与同步
  ├─ 右侧栏
  │   ├─ 账号安全
  │   ├─ 通用偏好
  │   └─ 数据导出
  └─ 底部保存动作条
```

- 桌面端采用 8/4 栅格。
- 小屏按单列堆叠。
- 保持左侧 240px 侧边栏布局。

### 2. 个人信息

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | text | 用户显示名称 |
| 职业身份 | text | 如高级 AI 产品经理 |
| 所在城市 | text | 用于岗位城市匹配 |
| 简介 | textarea | 个人求职背景描述 |

修改任一字段后，保存动作条进入“您有未保存的更改”状态。

### 3. 专业背景摘要

专业背景摘要来自 settings 和简历画像，可由 `resumes.ecosystem_json` 或 Career DNA 生成：

- 主要技能标签
- 目标行业
- 当前 AI 方向定位
- 最近一次简历分析时间

### 4. 通知与同步

| 设置项 | 说明 | 默认 |
|--------|------|:---:|
| 岗位风险提醒 | 发现高风险 JD 时推送通知 | 开 |
| 报告生成提醒 | 评估完成后发送站内消息 | 开 |
| 语言偏好同步 | 同步到报告与简历模板 | 关 |

通知设置应保存为 `settings.notification_preferences`。

### 5. 账号安全

账号安全区已经承接当前 owner 鉴权能力：

- 登录密码
- 当前登录账号
- 最近登录时间
- 退出登录

已实现密码修改和退出登录；手机号 / 邮箱换绑、短信验证、OAuth 不在当前范围。

### 6. 通用偏好

| 偏好 | 可选项 |
|------|--------|
| 主题模式 | 浅色 / 深色 / 自动 |
| 系统语言 | 简体中文 / English / 日本語 |

当前设计系统以浅色暖纸手帐为默认主题。深色与自动模式只保留控件入口，后续再接实际主题变量。

### 7. 数据导出

数据导出入口覆盖：

- 评估报告备份
- 简历档案备份
- 设置备份
- Tracker、Agent、面试、风险反馈等本地记录

当前生成 JSON 下载；不导出 API key、session token、密码明文或环境变量。

本地数据清除入口覆盖：
- 由用户勾选类别。
- 输入确认文本。
- 调用 `/api/data/clear`。
- 清除结果展示每类影响数量。

## 数据模型建议

```json
{
  "profile": {
    "name": "陈默",
    "role": "高级 AI 产品经理",
    "city": "上海，中国",
    "bio": "..."
  },
  "notifications": {
    "risk_alert": true,
    "report_ready": true,
    "language_sync": false
  },
  "preferences": {
    "theme": "light",
    "language": "zh-CN"
  }
}
```

建议按 key 写入 `settings` 表：

| key | value_json |
|-----|------------|
| `profile` | 个人信息 JSON |
| `notification_preferences` | 通知设置 JSON |
| `ui_preferences` | 主题与语言 JSON |

## API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/settings` | GET | 读取所有设置 |
| `/api/settings` | PUT | 批量保存设置 |
| `/api/export` | GET | 导出本地业务数据 JSON |
| `/api/data/clear` | POST | 按类别清除本地数据 |
| `/api/auth/me` | GET | 读取当前账号状态 |
| `/api/auth/password` | PATCH | 修改密码 |
| `/api/auth/logout` | POST | 退出登录 |

---

## 页面状态

| 状态 | 展示 |
|------|------|
| 默认态 | “当前设置已是最新”，保存按钮禁用 |
| 已修改 | “您有未保存的更改”，保存与取消可用 |
| 保存中 | 保存按钮 loading，表单禁用 |
| 保存成功 | 提示已保存，并回到默认态 |
| 保存失败 | 展示错误提示，保留用户改动 |

## 验收标准

- `/settings` 可从侧边栏进入，当前导航高亮正确。
- 修改任一表单或通知开关后，底部动作条能识别未保存更改。
- 点击取消后恢复默认值。
- 页面在桌面与窄屏下不出现文字溢出或控件重叠。
- 导出 JSON 不包含密钥、token 或密码明文。
- 清除数据必须显式选择类别并输入确认文本。
- 文案明确当前为本地 owner 设置管理，不承诺尚未实现的短信、OAuth、云同步或多用户安全操作。

# 12 — 简历管理页完整功能拆解

> 页面: `src/app/resume/page.tsx` | API: `/api/resume/parse /api/resume/match /api/resume/optimize` | 数据: SQLite resumes 表

---

## 当前实现补充（2026-06-11）

- `/resume` 的真实可见页面是 `ResumeWorkspace`，不是静态示例 `ResumeList.tsx`。
- 支持粘贴简历、加载 demo 简历、上传文件解析；解析后写入 SQLite `resumes`。
- 支持固定方向标签、默认简历、软归档；已归档简历不能继续匹配、设置默认、改方向或做 ATS 检查。
- Resume Judge v2 已接入：基于已保存简历和目标报告 / 手动 JD 生成优化建议，并保存 run 历史。
- ATS 检查已接入 `/api/resume/ats-check`，结果进入 `resume_optimization_runs`。
- 简历优化建议优先本地匹配与规则生成，可在模型可用时增强；不可用时必须显示 fallback。

## 功能拆解

### 1. 简历创建/导入

三种输入方式：

```
┌─────────────────────────────────────────────┐
│  📄 创建你的 AI 求职简历                       │
│                                             │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐ │
│  │ 📋 粘贴   │ │ 📁 上传   │ │ ✏️ 手动填写  │ │
│  │ 简历文本  │ │ PDF/Word │ │ 结构化表单   │ │
│  └──────────┘ └──────────┘ └──────────────┘ │
│                                             │
│  支持 Markdown / 纯文本 / PDF / Word 格式     │
└─────────────────────────────────────────────┘
```

- 粘贴：大文本域，直接粘贴简历内容
- 上传：竞赛版简化为支持纯文本和 Markdown（PDF/Word 解析为可选）
- 手动填写：结构化表单（姓名、当前岗位、工作年限、技能、项目经历、教育背景）

### 6. AI 优化建议

匹配完成后生成 3-5 条具体建议：

```markdown
## 简历优化建议

### 1. 项目描述强化
当前："搭建了企业 RAG 知识库系统"
建议："设计并实现企业级 RAG 知识库系统，采用 LangChain + Milvus 架构，日均处理 5000+ 查询，检索准确率 92%，支持多轮对话和上下文感知"

### 2. 关键词对齐
JD 要求 "Agent 开发经验"，你的简历有 LangChain 经验但未提及 Agent。
建议在项目经历中补充："基于 LangChain Agent 框架实现了自动化工单处理流程"

### 3. 量化成果
你的项目缺乏量化指标。建议为每个项目补充：
- 用户规模（多少人使用）
- 性能指标（QPS、延迟、准确率）
- 业务影响（效率提升%、成本降低%）
```

### 7. 多简历管理

- 支持创建多份简历（针对不同 AI 方向定制）
- 简历列表：名称 + 核心方向 + 更新时间
- 创建简历时选择方向："通用 AI" / "LLM 应用层" / "AI Infra"
- 支持设置默认简历
- 支持软归档，不物理删除数据
- 支持修改固定方向标签

### 8. Resume Judge / ATS

Resume Judge 面向“这份简历如何更贴合目标岗位”：
- 选择已保存简历。
- 选择历史报告或手动粘贴 JD。
- 选择优化模式与 effort。
- 生成问题诊断、优先级建议、可复制修改点。
- 保存历史 run，并支持用户反馈。

ATS 检查面向“格式和关键词是否适合筛选系统”：
- 计算 ATS 分。
- 展示检查项、问题和建议。
- 只读取用户选择的简历与目标 JD，不扫描整库。

## 页面状态

| 状态 | 展示 |
|------|------|
| **无简历** | 引导卡片："创建你的第一份 AI 求职简历" + 三种输入方式 |
| **解析中** | 骨架屏 + "AI 正在解析..." |
| **有简历** | 简历预览 + 生态位分析 + 操作按钮 |
| **匹配中** | JD 选择器 + "选择岗位开始匹配" |
| **匹配完成** | 匹配结果 + 缺口分析 + 优化建议 |
| **多简历** | 简历列表 + 切换管理 |
| **已归档** | 不参与匹配 / 默认 / ATS，可在列表中保留历史 |
| **Judge 中** | 优化建议生成中，保留当前输入 |
| **ATS 完成** | ATS 分、检查项和建议 |

## API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/resume/parse` | POST | 简历文本 → 结构化解析 |
| `/api/resume/match` | POST | 简历 + JD → 匹配分析 |
| `/api/resume/optimize` | POST | 简历 + JD → 优化建议 |
| `/api/resume/save` | POST | 保存简历 |
| `/api/resumes` | GET | 读取简历列表 |
| `/api/resumes/default` | GET | 读取默认简历 |
| `/api/resumes/[id]/default` | PATCH | 设置默认简历 |
| `/api/resumes/[id]/archive` | PATCH | 软归档简历 |
| `/api/resumes/[id]/direction` | PATCH | 更新简历方向 |
| `/api/resume/judge` | POST | Resume Judge 优化分析 |
| `/api/resume/judge/runs` | GET | Resume Judge 历史 |
| `/api/resume/judge/feedback` | POST | 记录优化建议反馈 |
| `/api/resume/ats-check` | POST | ATS 检查 |

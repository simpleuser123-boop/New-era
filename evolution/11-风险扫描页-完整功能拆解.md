# 11 — 风险扫描页完整功能拆解

> 页面: `src/app/risks/page.tsx` | API: `/api/scan-risks` | 数据: 本地规则引擎 + LLM 语义补充

---

## 当前实现补充（2026-06-11）

- 页面标题已收口为“AI 岗真假识别”，包含“单岗验真”和“真假对决”两个页面内模式。
- 单岗验真继续调用 `POST /api/scan-risks`，扫描结果内嵌 HR 核实问题组件。
- 真假对决使用预置组合，两侧调用同一个 `/api/scan-risks` 契约；不抓取外部招聘平台，不保存对决结果。
- 识别质量面板调用 `GET /api/risk-benchmark`，只读展示本地 benchmark 指标、样本边界和规则版本。
- HR 核实问题调用 `POST /api/risk-verification-questions`，只生成可复制问题，不自动发送、不保存联系人或外部消息。
- 报告页风险反馈写入 `risk_feedback`，用于补充识别质量闭环。

## 功能拆解

### 1. JD 文本输入

和评估页共用同一个输入组件，但更简洁：
- 纯文本输入，不提供 URL 抓取
- 最小 30 字符即可扫描
- placeholder："粘贴 JD 文本，检测是否为假 AI 岗..."

### 2. 一键风险扫描

```
用户点击「扫描风险」
    │
    ▼
┌─────────────────────────────┐
│ Layer 1: 正则匹配 (20+ 规则) │  ← 毫秒级
│ → 命中: "零基础转行 AI"      │
│ → 命中: "培训费从工资扣"     │
├─────────────────────────────┤
│ Layer 2: 字典子串匹配        │  ← 毫秒级
│ → 无 AI 技术栈关键词         │
│ → 命中: "亲自带教"           │
├─────────────────────────────┤
│ Layer 3: LLM 语义补充        │  ← 1-2 秒
│ → "全方位 AI 成长支持"       │
│   (培训贷变体表述)           │
└─────────────┬───────────────┘
              ▼
     合并去重 → 加权计分 → 风险等级
```

### 3. 风险等级总览

```
┌──────────────────────────────────────────────┐
│                                              │
│         🔴 假 AI 岗（严重风险）                 │
│                                              │
│     检测到 5 个风险信号，综合评分强制降至 1.0     │
│                                              │
│    ┌─────────────────────────────────────┐   │
│    │ critical: 2    high: 2    medium: 1 │   │
│    └─────────────────────────────────────┘   │
│                                              │
│    ⚠️ 建议：不投递。疑似 AI 培训公司            │
│    伪装招聘，建议核实公司资质。                  │
└──────────────────────────────────────────────┘
```

四级风险等级展示：
- 🔴 虚假 AI 岗：红色背景卡片，大号警告图标
- 🔴 高风险：红色边框卡片
- 🟡 中风险：黄色边框卡片
- 🟢 低风险：绿色边框卡片

### 4. 风险信号列表

每条风险信号展开显示：

```
┌─────────────────────────────────────────────────────────┐
│ ⚠️ critical  培训贷 / 招转培                               │
│                                                         │
│ 📝 JD 原文："公司提供全方位 AI 成长支持，零基础也能快速上手，  │
│           先培训后上岗，培训期间有生活补贴，转正后薪资上不封顶"  │
│                                                         │
│ 📖 解读：此为典型"招转培"模式 — 以招聘为名诱导参加培训，       │
│         常见套路：先培训 → 签培训贷 → 无法转正 → 背负贷款。    │
│         人社部 2026 年已专项打击此类骗局。                    │
│                                                         │
│ 🔍 检测来源：Layer 1 正则 + Layer 3 LLM 语义确认            │
└─────────────────────────────────────────────────────────┘
```

- 原文引用中匹配到的关键词用红色高亮标记
- 每条信号附带解读和建议
- 标注检测来源（正则 / 字典 / LLM）

### 5. 真/假 AI 岗对决

竞赛 Demo 的核心视觉亮点：

```
┌──────────────────────┐  ┌──────────────────────┐
│  真 AI 岗             │  │  假 AI 岗             │
│                      │  │                      │
│  🟢 低风险           │  │  🔴 假 AI 岗          │
│  评分: 4.2/5         │  │  评分: 1.0/5          │
│                      │  │                      │
│  AI 技术栈:           │  │  AI 技术栈:           │
│  ✅ LangChain        │  │  ❌ 无 AI 框架要求     │
│  ✅ RAG              │  │  ❌ 无 AI 工具链       │
│  ✅ Agent            │  │                      │
│  ✅ PyTorch          │  │  风险信号:             │
│                      │  │  ⚠️ 零基础转行         │
│  风险信号: 无         │  │  ⚠️ 培训费从工资扣    │
│                      │  │  ⚠️ 无 AI 技术栈      │
│                      │  │  ⚠️ 薪资上不封顶      │
```

并排两个卡片，视觉差异一目了然。当前实现通过页面内模式切换进入“真假对决”，并明确标注两侧使用同一套扫描 API。

### 6. 识别质量看板

识别质量面板展示：
- 本地 benchmark 样本数量。
- 当前风险规则版本。
- 准确率、召回率等质量指标。
- 样本边界说明，避免把本地 benchmark 误读为全网质量承诺。

该接口只读，不写 SQLite，不触发模型调用。

### 7. HR 核实问题

扫描出风险后，页面可把风险证据转成核实问题：
- 按风险优先级排序。
- 支持逐条复制和全部复制。
- 问题仅供用户人工核实，不自动发送消息。
- API 不写 reports、applications、settings、sessions 等长期业务表。

### 8. 假 AI 岗知识库

页面底部展示常见骗术模式：

| 骗术类型 | 典型话术 | 识别方法 |
|---------|---------|---------|
| 培训贷 | "零基础转行 AI，先培训后上岗" | 出现"培训费""先学后上岗"等关键词 |
| 传统开发包装 | JD 标题"AI 工程师"，内容全是 Java CRUD | 技术栈白名单对不上 |
| 过度承诺 | "参与大模型训练"（实际做数据标注） | 要求与岗位不符 |
| AI 培训公司伪装 | 公司名含"教育"但岗位是"开发工程师" | 公司名检测 |

## 页面状态

| 状态 | 展示 |
|------|------|
| **初始态** | 输入框 + 预设 Demo 对比卡片 + 知识库 |
| **扫描中** | 三层进度指示（正则 ✓ → 字典 ⏳ → LLM ⏳） |
| **有风险** | 红色风险总览 + 风险信号列表 |
| **无风险** | 绿色"未检测到风险信号" + 真 AI 岗确认 |

| **真假对决** | 双列扫描结果 + 差异摘要 |
| **识别质量不可用** | 明确错误提示，不暴露本机绝对路径 |

## API 端点

|------|------|------|

| `/api/risk-verification-questions` | POST | 生成 HR 核实问题 |

# 14 — 登录注册页完整功能拆解

> 页面: `src/app/auth/page.tsx` | 组件: `AuthCard` | API: `/api/auth/register /api/auth/login /api/auth/logout /api/auth/me /api/auth/password`

---

## 范围说明

登录/注册页已经接入 local-first owner 鉴权。当前产品仍保持单用户、本地数据场景，但不再是纯前端演示态。

当前边界：
- 实现 owner 注册、登录、登出、会话读取和密码修改。
- 密码只保存 hash / salt / 参数，不保存明文。
- 不实现短信验证码、第三方 OAuth、多租户、云同步。
- 受保护页面和 API 通过 session cookie 校验。

## 功能拆解

### 1. 页面定位

登录/注册页不参与核心演示闭环，但用于：

- 让评委看到产品具备正式应用的入口形态。
- 和设置页、账号安全等支撑页面保持概念一致。
- 后续接真实用户体系时，前端结构可以复用。

### 2. 登录/注册模式切换

```text
登录页
  ├─ 登录 Tab: 进入我的职场画布
  └─ 注册 Tab: 创建职场账户
```

- 默认展示登录模式。
- 切换后展示对应字段。
- 提交按钮文案随模式变化。

### 3. 注册字段

```text
注册
  ├─ 显示名
  ├─ 邮箱或手机号
  ├─ 密码
  └─ 确认密码
```

- 当前系统是单 owner 模式。
- 已存在 owner 时不再创建第二个 owner。
- 输入通过 Zod 校验后写入 `users`。

### 4. 登录字段

```text
登录
  ├─ 账号：邮箱或手机号
  └─ 密码
```

- 登录成功后写入 `auth_sessions`，前端跳转回受保护页面或首页。
- 登录失败只给通用错误，不暴露账号是否存在。

### 5. 会话与受保护访问

- 页面侧使用 `requireProductPageAccess`。
- API 侧使用 `requireApiAuth`。
- `/api/auth/me` 返回当前登录态和 owner 信息。
- session 过期、撤销或不存在时，引导重新登录。

### 6. 密码修改与登出

密码修改入口在 `/settings`：
- 校验当前密码。
- 更新 password hash / salt / params。
- 保留当前 session，撤销其他 session。

登出入口在 `/settings`：
- 弹出确认对话框。
- 撤销当前 session。
- 跳转到登录页。

### 7. 提交状态

| 状态 | 展示 |
|------|------|
| 初始态 | 主按钮展示当前模式动作 |
| 处理中 | 按钮禁用，展示加载图标与“处理中...” |
| 验证成功 | 按钮展示成功图标与“验证成功” |
| 失败态 | 展示字段级错误或顶部错误提示 |

### 8. 第三方登录边界

当前不实现微信、GitHub 或短信验证码登录。页面文案不得暗示这些能力已经可用。

## API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/auth/register` | POST | 创建 owner 账号 |
| `/api/auth/login` | POST | 密码登录 |
| `/api/auth/logout` | POST | 撤销当前 session |
| `/api/auth/me` | GET | 读取当前登录态 |
| `/api/auth/password` | PATCH | 修改密码并撤销其他 session |

---

## 验收标准

- `/auth` 可直接访问，移动端与桌面端均不溢出。
- 登录/注册 Tab 切换无页面跳动。
- 注册后可进入受保护页面。
- 登录后刷新页面仍保持会话。
- 登出后受保护页面会跳回登录。
- 密码修改后旧密码不可登录，新密码可登录。
- 页面文案不暗示当前已接入真实短信、第三方登录或多用户 SaaS。

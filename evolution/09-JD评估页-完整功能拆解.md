# 09 — JD 评估页完整功能拆解

> 页面: `src/app/evaluate/page.tsx` | API: `/api/evaluate /api/evaluate/stream /api/jd/fetch` | 数据: SQLite reports 表

---

## 当前实现补充（2026-06-11）

- `/evaluate` 属于受保护产品页面，需要 owner 登录。
- 评估完成后会保存报告，并可进入 `/evaluate/report?reportId={id}` 读取 SQLite 中的真实报告。
- `/evaluate/report` 保留无 `reportId` 的 demo / 草稿报告能力，但真实历史入口使用 `reportId`。
- 报告页可继续执行：打印导出、复制报告摘要、加入本地投递清单、创建面试准备入口、风险反馈。
- `/api/jd/fetch` 只抓取公开页面正文；不接招聘平台授权，不读取用户登录态。

| # | 功能 | 实现 | 数据源 |

## 功能拆解

### 1. JD 文本输入

```
┌─────────────────────────────────────────────┐
│  粘贴 JD 文本，或输入招聘页面 URL             │
│                                             │
│  ┌─────────────────────────────────────────┐│
│  │                                       ││
│  │  （大文本输入区域，最少 50 字符）          ││
│  │                                       ││
│  └─────────────────────────────────────────┘│
│                                             │
│  已输入 342 字符  [开始评估]                  │
└─────────────────────────────────────────────┘
```

- 最小字符数：50（少于 50 字符按钮灰色不可点击）
- 实时字符计数
- placeholder 提示："粘贴 JD 文本，例如：岗位职责... 任职要求..."
- 支持自动识别 URL（粘贴后 1 秒内自动判定）

### 2. JD URL 抓取

```
用户粘贴 URL
    │
    ▼
前端正则匹配 URL 格式
    │
    ▼
POST /api/jd/fetch { url: "..." }
    │
    ▼
服务端 Cheerio 抓取页面正文
    ├─ 去除 nav/footer/script/style
    ├─ 提取主体文本
    └─ 返回 { text, title }
    │
    ▼
填充到输入框，用户可编辑后再评估
```

- 支持的平台：Boss直聘、拉勾、猎聘、51job（公开页面）
- 抓取失败：提示"抓取失败，请手动粘贴 JD 文本"
- 加载态：输入框显示"正在抓取 JD 内容..." + 旋转动画
- 有 title 时：在输入框上方显示"已识别：{岗位名称}"

### 3. 简历上下文加载

评估页自动检测用户是否已保存简历：
- 有简历 → 在输入框下方显示"将自动使用已保存的简历进行匹配" + 简历名称
- 无简历 → 显示"建议先添加简历，以获得更精准的技能匹配" + 跳转链接

### 4. 流式评估展示

```
SSE 事件流 → 前端逐维渲染

┌─────────────────────────────────────────────┐
│ 🔍 正在评估：深度求索 — AI 产品经理            │
│                                             │
│ ████████████████████████░░░░ 83%             │
│                                             │
│ ✅ A 岗位定位      已完成                      │
│ ✅ B 技术栈匹配    已完成                      │
│ 🔄 C 经验要求      评估中...                   │
│ ⏳ D 成长价值      等待中                      │
│ ⏳ E 薪酬竞争力    等待中                      │
│ ⏳ F 风险合规      等待中                      │
└─────────────────────────────────────────────┘
```

每个维度完成后：
- 图标从 ⏳ 变为 ✅
- 进度条推进
- 该维度的简要结果内联展示（可折叠）

全部完成后 → 跳转到评估报告页 `/evaluate/report?reportId={id}`；如果只是本地草稿或 demo，则使用无 `reportId` 的报告视图。

### 5. 快速评估（预设 Demo）

竞赛 Demo 关键功能 — 预置 5-8 条仿真 JD 供演示：

| # | 案例 | 类型 | 演示目的 |
|---|------|------|---------|
| 1 | DeepSeek — AI 产品经理 | 真 AI 岗 | 展示完整 6 维评估 + 4.5+ 高分 |
| 2 | 智谱 AI — 大模型应用开发 | 真 AI 岗 | 展示技术栈深度匹配 |
| 3 | 某大厂 — AI 算法工程师 | 边界岗 | 展示中风险信号 + 3.5 分 |
| 4 | 某培训机构 — AI 讲师 | 假 AI 岗 | 展示风险引擎拦截 + 2.1 分 |
| 5 | 某初创 — LLM Agent 开发 | 真 AI 岗 | 展示高成长价值 + 低薪酬 |
| 6 | 某外包公司 — AI 开发 | 假 AI 岗 | 展示传统开发伪装检测 |

点击预设案例卡片 → 自动填入 JD 文本 → 一键评估。

### 6. 评估历史入口

页面底部：最近 3 条评估记录 + "查看全部 →" 链接。

## 页面状态

| 状态 | 展示 |
|------|------|
| **初始态** | 空输入框 + 预设 Demo 卡片网格（5-8 个） |
| **输入中** | 输入框有内容，字符计数，按钮高亮 |
| **URL 抓取中** | 输入框禁用 + "正在抓取..." 提示 |
| **抓取失败** | 红色提示 + 引导手动粘贴 |
| **评估中** | 流式进度面板，逐维渲染 |
| **评估完成** | 自动跳转报告页 |
| **评估失败** | 错误提示 + 重试按钮 |
| **无简历** | 顶部横幅提示"添加简历获得更精准匹配" |

## API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/evaluate` | POST | 同步评估（返回完整报告 JSON） |
| `/api/evaluate/stream` | POST | SSE 流式评估 |
| `/api/jd/fetch` | POST | JD URL 抓取 |
| `/api/reports` | GET | 评估历史列表 |
| `/api/reports` | POST | 保存评估报告 |
| `/api/reports/[id]` | GET | 读取单份报告 |

## 2. 表结构

> 当前表结构以 `src/lib/server-db.ts` 为准。本文记录当前业务含义和迁移边界，不替代代码中的 SQL。

### 2.1 reports — 评估报告

```sql
CREATE TABLE IF NOT EXISTS reports (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  company     TEXT NOT NULL,
  role        TEXT NOT NULL,
  jd_text     TEXT NOT NULL,
  score_json  TEXT NOT NULL DEFAULT '{}',    -- A-F 6维分数 JSON
  risks_json  TEXT NOT NULL DEFAULT '[]',    -- 风险信号数组 JSON
  report_json TEXT NOT NULL DEFAULT '{}',    -- 完整报告 JSON
  created_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX IF NOT EXISTS idx_reports_created ON reports(created_at DESC);
```

### 2.2 resumes — 简历数据

```sql
CREATE TABLE IF NOT EXISTS resumes (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  name            TEXT NOT NULL DEFAULT '未命名',
  content_json    TEXT NOT NULL DEFAULT '{}',   -- 简历结构化数据
  skills_json     TEXT NOT NULL DEFAULT '[]',   -- 技能列表 JSON
  ecosystem_json  TEXT NOT NULL DEFAULT '{}',   -- 生态位向量 JSON
  created_at      TEXT NOT NULL DEFAULT (datetime('now')),
  direction       TEXT NOT NULL DEFAULT 'general',
  is_default      INTEGER NOT NULL DEFAULT 0,
  archived_at     TEXT,
  updated_at      TEXT NOT NULL DEFAULT (datetime('now'))
);
```

当前实现补充：
- `direction` 使用固定 AI 方向标签，用于简历方向和岗位方向对比。
- `is_default` 有唯一默认简历约束。
- `archived_at` 代表软归档；不做物理删除。

### 2.3 settings — 用户设置

```sql
CREATE TABLE IF NOT EXISTS settings (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  key         TEXT NOT NULL UNIQUE,
  value_json  TEXT NOT NULL DEFAULT '{}',
  updated_at  TEXT NOT NULL DEFAULT (datetime('now'))
);
```

当前常用 key：
- `profile`
- `notification_preferences`
- `ui_preferences`
- `career_profile`
- `product_guide`

### 2.4 applications / application_events — 本地投递 Tracker

`applications` 保存用户手动维护的本地投递记录，不代表平台投递成功：
- 公司、岗位、来源 URL、关联报告、关联简历。
- 状态：待投递、已投递、面试中、offer、已拒绝、已放弃。
- 优先级、渠道、下一次跟进时间、投递 / 面试 / offer / 拒绝 / 放弃时间。
- 标签、备注、更新时间、风险等级。

`application_events` 保存投递时间线，记录状态变化和用户手动新增事件。

### 2.5 users / auth_sessions / auth_events — owner 鉴权

当前实现为单机 owner 账号：
- `users` 保存 email / phone / display_name / 密码 hash / salt / 参数 / role。
- `auth_sessions` 保存 token hash、过期时间、last_seen、revoked_at。
- `auth_events` 记录 owner 创建、登录、登出、密码变更等事件。

边界：
- 不是多租户 SaaS。
- 不实现短信验证码、第三方 OAuth 或云同步。

### 2.6 agent_action_proposals / agent_sessions / agent_memory_items — Agent 状态

- `agent_sessions` 保存 `/assistant` 会话标题、消息 JSON、memory digest。
- `agent_memory_items` 保存可控记忆，支持软删除。
- `agent_action_proposals` 保存 Agent 生成的待确认动作。

边界：
- Query 工具只读。
- Action 工具只能创建 proposal；确认前不得直接改 `applications`、`settings`、`interview_sessions` 等业务表。

### 2.7 interview_sessions / interview_answers / stories — 面试教练

- `interview_sessions` 保存目标报告、简历、标题、模式、问题列表和更新时间。
- `interview_answers` 保存回答文本和评分 JSON。
- `stories` 保存 STAR 故事库条目。

### 2.8 resume_optimization_runs / optimization_feedback — Resume Judge

- `resume_optimization_runs` 保存 Resume Judge / ATS 检查结果、目标报告或 JD、模式和 effort。
- `optimization_feedback` 保存用户对建议的采纳 / 拒绝 / 备注。

### 2.9 risk_feedback — 风险识别反馈

保存报告页和 benchmark 相关风险反馈：
- 规则版本、原始风险等级、用户标注、反馈类型、风险信号、备注。
- 支持按 `report_id`、`benchmark_jd_id`、`risk_rule_version` 查询。

## 3. 写入路径

**唯一写入路径：server-db.ts → SQLite**

```typescript
// src/lib/server-db.ts
import Database from 'better-sqlite3';

let db: Database.Database;

export function getDb(): Database.Database {
  if (!db) {
    db = new Database('data/new-era.db');
    db.pragma('journal_mode = WAL');
    db.pragma('busy_timeout = 5000');
    db.pragma('foreign_keys = ON');
    autoRunMigrations(db);
  }
  return db;
}
```

## 4. 并发与完整性

| 机制 | 实现 |
|------|------|
| WAL 模式 | `pragma journal_mode=WAL` — 读写并发安全 |
| 繁忙超时 | `pragma busy_timeout=5000` — 等 5 秒 |
| 输入校验 | zod schema 校验所有写入数据 |
| 自动建表 | 启动时 `autoRunMigrations()` 执行建表 SQL |
| additive migration | `PRAGMA table_info(...)` + guarded `ALTER TABLE` |
| 导出边界 | `/api/export` 导出本地业务数据，不导出环境变量密钥 |
| 清除边界 | `/api/data/clear` 只清除用户确认的本地类别 |

## 5. 当前边界说明

| 能力 | 当前做法 | 边界 |
|------|----------|------|
| 本地数据 | SQLite 单文件 | 不做云同步和多用户共享 |
| 鉴权 | 单 owner 账号 + session cookie | 不做 OAuth / 短信 / 多租户 |
| Agent 写入 | 待确认 proposal | 不允许 LLM 直接写业务表 |
| 投递管理 | 用户手动维护 Tracker | 不代替用户投递，不接招聘平台 |
| 数据导出 | JSON 导出本地记录 | 不导出 API key / token / 环境变量 |
| 数据清除 | 明确选择类别 + 确认文本 | 不做隐藏自动清除 |
| 风险识别 | 本地规则、benchmark、反馈闭环 | 不声称能穷尽所有招聘风险 |

## 2. 技术选型

### 2.1 前端框架

| 组件 | 选型 | 理由 |
|------|------|------|
| 框架 | **Next.js 16 (Turbopack) + React 19** | App Router, Server Components, 全栈一体化 |
| 样式 | **Tailwind CSS 4 + CSS 变量** | globals.css 定义设计令牌 |
| 动画 | **Framer Motion** | 页面过渡、卡片入场动画 |
| 图标 | **Lucide React** | 轻量、一致、Tree-shakable |
| 字体 | **Noto Sans SC** | 中文正文阅读舒适 |

### 2.2 AI 模型

| 组件 | 选型 | 理由 |
|------|------|------|
| 主模型 | **DeepSeek API** | 中文 JD、简历、面试与助手场景优先 |
| 接入方式 | 服务端环境变量 / 中转站 API | 密钥不写入导出数据和 settings |
| 降级策略 | 规则 / 本地算法 / demo fallback | 无 key 或模型不可用时核心演示仍可运行 |

### 2.3 数据层

| 组件 | 选型 | 理由 |
|------|------|------|
| 数据库 | **SQLite (WAL模式)** | 单文件、零运维、适合本地优先产品 |
| 数据库驱动 | **better-sqlite3** | 同步 API，简化逻辑 |
| Schema 管理 | 声明式建表 SQL + additive migration | 兼容已有本地库，不删除用户数据 |

### 2.4 后端工具链

| 组件 | 选型 | 理由 |
|------|------|------|
| 运行时 | **Node.js 22** | Next.js 默认 |
| JD 抓取 | **Cheerio + fetch** | 轻量 DOM 解析 |
| PDF 生成 | **浏览器打印** | 报告页通过 `window.print()` 导出 |
| 输入校验 | **zod** | 类型安全 + 运行时校验 |

## 3. 关键技术决策

### 3.1 数据层：SQLite 单文件

**选择 SQLite 而非 PostgreSQL/MySQL。**

理由：
- 本地优先，零运维负担
- 单文件方便打包展示和迁移
- WAL 模式支持读写并发
- `better-sqlite3` 同步 API 简化逻辑
- additive migration 保护既有用户数据

### 3.2 AI 模型：DeepSeek + 本地降级

**选择 DeepSeek 作为主要模型能力，但关键链路必须有本地可解释降级。**

理由：
- JD 风险识别、简历匹配、Analytics 和部分 Agent 查询可以用本地规则/数据库完成。
- 无 API key、模型超时或中转站不可用时，页面应明确标记 fallback，不伪装成实时模型结果。
- 不做多 provider 复杂调度，避免偏离 local-first 产品边界。

### 3.3 评估引擎：SSE 流式输出

**选择 SSE 流式而非一次性返回。**

理由：
- 评估过程长（6 个维度逐维输出），流式给用户"AI 正在工作"的体感

- 每个维度完成后前端实时渲染，不需要转圈等待

  

### 3.4 风险检测：确定性与 LLM 结合

**选择 正则+字典（确定性）+ LLM 语义补充，而非纯 LLM。**

理由：
- 假 AI 岗检测有大量确定性规则可穷举（"零基础""培训后上岗"等关键词）
- 确定性规则零成本、零幻觉、可审计
- LLM 仅用于覆盖变体表述（如"我们提供全方位 AI 成长支持"）

### 3.5 前端设计：暖纸手帐风格

**选择暖色纸张感设计，而非科技感暗色主题。**

理由：

- 暖色调降低求职焦虑感
- 区别于所有"招聘 App 蓝"的刻板印象
- 在竞赛中视觉差异化明显

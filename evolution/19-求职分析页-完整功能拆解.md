# 19 — 求职分析页完整功能拆解

> 页面: `src/app/analytics/page.tsx` | 组件: `AnalyticsPage / AnalyticsWorkspace` | API: `/api/analytics` | 数据: reports / resumes / applications / settings.career_profile

---

## 当前实现补充（2026-06-11）

求职分析是个人本地数据复盘，不是平台级统计。它聚合本地报告、简历、Tracker 和 Career DNA，并在样本不足时展示 caveat，避免把少量记录误读为稳定转化率。

## 功能拆解

### 1. 页面定位

Analytics 用于回答：
- 当前求职漏斗是否健康？
- 哪些方向评估多、推进少？
- 哪些风险信号反复出现？
- 默认简历和目标岗位的技能缺口是什么？
- 当前推进岗位是否偏离 Career DNA？

### 2. 样本量与 caveat

页面必须展示样本量：
- 报告数量。
- 简历数量。
- 投递记录数量。
- 活跃投递数量。

当记录较少时，页面展示 caveat：
- 结论只适合个人复盘。
- 不适合作为稳定转化率判断。
- 未设置默认简历或 Career DNA 时，相关模块会受限。

### 3. 求职漏斗

漏斗口径：

| 阶段 | 来源 |
|------|------|
| 已评估 | reports |
| 待投递 / 已投递 / 面试中 / offer / 拒绝 / 放弃 | applications.status |

### 4. 方向分析

方向分析读取报告与 Tracker：
- 聚合 JD 所属 AI 方向。
- 对比不同方向的推进状态。
- 用本地记录说明当前投入是否集中。

### 5. Career DNA 匹配

读取 `settings.career_profile`：
- 目标方向是否匹配。
- 城市和工作模式是否存在偏离。
- 输出 matched / warning 统计和提示。

未配置 Career DNA 时，页面应说明无法判断画像匹配。

### 6. 风险信号与技能缺口

风险信号：
- 聚合 reports 中的风险信号。
- 展示高频风险类型和出现次数。

技能缺口：
- 优先使用默认简历。
- 没有默认简历时使用最近简历，并显示 caveat。
- 无可用简历时，模块显示空状态。

### 7. 逾期跟进

基于 Tracker：
- 找出下一次跟进时间已过且仍在推进中的记录。
- 展示公司、岗位、状态和逾期天数。
- 不自动创建提醒或外部通知。

## 页面状态

| 状态 | 展示 |
|------|------|
| 加载中 | 分析骨架屏 |
| 有数据 | 漏斗、方向、画像、风险、技能缺口 |
| 空数据 | 引导先完成评估和投递记录 |
| 样本不足 | 展示 caveat |
| 加载失败 | 错误提示 + 重试 |

## 验收标准

- `/analytics` 可从侧边栏进入，导航高亮正确。
- 本地有 reports、applications、career_profile 时，页面显示对应分析。
- 无数据时页面不崩溃，并给出下一步入口。
- 样本不足、无默认简历、无 Career DNA 时有明确 caveat。
- 页面不声称数据来自招聘平台或全网统计。

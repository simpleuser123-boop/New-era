# 21 — 面试教练页完整功能拆解

> 页面: `src/app/interview/page.tsx` | 组件: `InterviewPage / InterviewWorkspace` | API: `/api/interview/sessions/* /api/stories` | 数据: SQLite `interview_sessions` / `interview_answers` / `stories`

---

## 当前实现补充（2026-06-11）

面试教练基于本地报告和简历上下文生成面试准备材料，支持创建会话、生成问题、提交回答、评分反馈和维护 STAR 故事库。它不自动联系 HR，不发送外部消息。

## 功能拆解

### 1. 页面定位

面试教练把“岗位评估结果”转成“面试准备动作”：
- 从报告中提取岗位重点。
- 从简历中提取可讲项目。
- 生成技术、业务、风险核实、行为面试问题。
- 让用户练习回答并获得反馈。

### 2. 面试会话

会话字段：

| 字段 | 说明 |
|------|------|
| reportId | 可选，关联评估报告 |
| resumeId | 可选，关联简历 |
| title | 会话标题 |
| mode | mixed / technical / behavioral 等 |
| questions_json | 当前问题列表 |
| created_at / updated_at | 时间戳 |

### 3. 创建会话

入口：
- `/interview` 页面内创建。
- 报告页带 `reportId` 跳转。
- Agent action proposal 经用户确认后创建。

创建后写入 `interview_sessions`。

### 4. 问题生成

调用：

```text
POST /api/interview/sessions/[id]/questions
```

问题类型：
- 岗位理解。
- AI 技术栈。
- 项目深挖。
- 风险核实。
- 行为面试。

### 5. 回答与评分

回答提交调用：

```text
POST /api/interview/sessions/[id]/answers
```

保存内容：
- question_id
- answer_text
- score_json
- created_at

评分反馈应给出：
- 优势。
- 缺口。
- 可补充证据。
- 下一轮优化建议。

### 6. STAR 故事库

`/api/stories` 支持：
- 读取故事库。
- 新增 STAR 故事。

故事字段：
- title
- situation
- task
- action
- result
- tags

### 7. 与 HR 核实问题的边界

风险扫描页的 HR 核实问题用于判断岗位真实性；面试教练用于准备面试回答。两者可以共享报告和风险上下文，但不应混成同一个自动发送动作。

## 页面状态

| 状态 | 展示 |
|------|------|
| 加载中 | 会话列表 / 当前会话骨架 |
| 无会话 | 引导从报告创建面试准备 |
| 有会话 | 问题、回答、反馈、故事库 |
| 生成问题中 | 按钮 loading，保留当前上下文 |
| 回答评分中 | 回答区禁用，显示评分中 |
| 保存失败 | 错误提示，保留回答草稿 |

## API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/interview/sessions` | GET/POST | 读取 / 创建面试会话 |
| `/api/interview/sessions/[id]` | GET | 读取会话详情 |
| `/api/interview/sessions/[id]/questions` | POST | 生成问题 |
| `/api/interview/sessions/[id]/answers` | POST | 提交回答并评分 |
| `/api/stories` | GET/POST | 读取 / 新增 STAR 故事 |

---

## 验收标准

- `/interview` 可从侧边栏进入，导航高亮正确。
- 从报告页进入时，能携带 `reportId` 创建或预填面试准备。
- 会话创建、问题生成、回答提交后刷新不丢失。
- STAR 故事可保存并在后续会话中复用。
- 页面不承诺自动联系 HR、自动发送问题或替用户完成面试沟通。

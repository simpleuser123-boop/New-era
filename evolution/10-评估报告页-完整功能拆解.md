# 10 — 评估报告页完整功能拆解

> 页面: `src/app/evaluate/report/page.tsx` | API: `/api/reports/[id]` | 数据: SQLite reports 表

---

## 当前实现补充（2026-06-11）

- 报告页支持 `?reportId={id}` 读取真实 SQLite 报告；无 `reportId` 时保留草稿 / demo 视图。
- 报告可通过 `window.print()` 打印导出，不再规划 Playwright 服务端 PDF 作为当前主路径。
- 报告摘要可复制到剪贴板，用于替代早期“公开分享”设想。
- 报告页可创建本地投递记录，调用 `POST /api/applications`，重复加入会返回明确 duplicate 状态。
- 风险区支持用户反馈，写入 `risk_feedback`，用于真假 AI 岗识别质量闭环。
- 有真实 `reportId` 时，可从报告进入 `/interview?reportId={id}` 创建面试准备。

## 功能清单

| # | 功能 | 实现 | 数据源 |
|---|------|------|--------|
| 1 | 报告头部摘要 | 公司+岗位+日期+综合评分 | SQLite |
| 2 | 风险提示横幅 | 置顶风险信号表格（有风险时） | SQLite risks_json |
| 3 | A-F 六维折叠面板 | 逐维展开详细信息 | SQLite report_json |
| 4 | 技术关键词云 | 15-20 个提取关键词 | SQLite |
| 5 | 简历匹配对比 | JD要求 vs 简历技能的对照表 | SQLite + resume |
| 6 | 导出 / 复制 | 浏览器打印 + 复制报告摘要 | 前端渲染 |
| 7 | 后续动作 | 加入投递清单 / 创建面试准备 / 风险反馈 | API |

## 功能拆解

### 1. 报告头部摘要

```
┌──────────────────────────────────────────────────────────────┐
│  ← 返回评估列表                                               │
│                                                              │
│  深度求索 DeepSeek                         2026-05-28         │
│  AI 产品经理 · LLM 应用方向                                    │
│                                                              │
│  ┌──────┐                                                    │
│  │ 4.2  │  综合评分                    🟢 低风险              │
│  │ /5.0 │  强烈建议投递                                      │
│  └──────┘                                                    │
│                                                              │
│  AI 领域：LLM 应用层    团队阶段：成熟产品迭代                    │
└──────────────────────────────────────────────────────────────┘
```

- 评分用大号琥珀色数字 + 背景光晕
- 分数对应的建议文案（4.5+ 强烈建议 / 4.0-4.4 建议 / ...）
- 风险等级用彩色 Badge

### 2. 风险提示横幅

仅当检测到风险信号时显示，置顶：

```
┌──────────────────────────────────────────────────────────────┐
│  ⚠️ 风险提示（中风险）                                         │
│                                                              │
│  | 信号 | JD 原文 | 风险等级 | 说明 |                         │
│  |------|---------|---------|------|                         │
│  | 弹性工作制 | "实行弹性工作制度" | 中 | 可能为无加班费信号 |     │
│  | 亲自带 | "由资深工程师亲自带教" | 中 | 可能无培训体系 |      │
│                                                              │
│  综合评分已从 4.2 调整为 3.8（受风险因素影响）                    │
└──────────────────────────────────────────────────────────────┘
```

- 风险等级配色：绿低 / 黄中 / 红高
- 每条风险信号附带 JD 原文引用
- 显示评分调整幅度

### 3. A-F 六维折叠面板

每个维度一个可折叠面板：

```
┌─ A 岗位定位 ───────────────────────────── 得分 4.5/5 ─┐
│                                                        │
│  AI 细分领域：LLM 应用层                                │
│  公司 AI 战略位置：核心 AI 业务（DeepSeek 自研大模型）    │
│  团队阶段：成熟产品迭代，有明确的产品规划和用户基础        │
│  一句话总结：国内顶尖 AI 公司的核心应用层岗位               │
└────────────────────────────────────────────────────────┘

┌─ B 技术栈匹配 ─────────────────────────── 得分 4.0/5 ─┐
│                                                        │
│  | JD 要求 | 简历匹配项 | 匹配度 | 说明 |               │
│  |---------|----------|--------|------|               │
│  | LangChain/LlamaIndex | 项目 #2 | 强 | RAG 系统搭建 | │
│  | Agent 开发 | — | 缺口 | 有基础可快速上手 |           │
│  | Prompt Engineering | 自评 | 强 | 日常高频使用 |     │
└────────────────────────────────────────────────────────┘

┌─ C 经验要求 ───────────────────────────── 得分 3.5/5 ─┐
┌─ D 成长价值 ───────────────────────────── 得分 4.5/5 ─┐
┌─ E 薪酬竞争力 ─────────────────────────── 得分 4.0/5 ─┐
┌─ F 风险合规 ───────────────────────────── 🟢 低风险 ──┘
```

- 默认展开 A 和 B（最重要的两个维度）
- 其余折叠，点击展开
- B 板块内的匹配度用颜色标识：强(绿) / 可弥补(黄) / 缺口(红)

### 6. 导出与复制

- 按钮位置：报告头部右侧
- 点击打印 → 浏览器打印对话框
- 点击复制 → 复制报告摘要到剪贴板
- 导出格式：保持报告样式，A4 纸张

### 7. 后续动作

- `加入投递清单`：仅写入本地 Tracker，不代表平台投递。
- `准备面试`：带 `reportId` 进入面试教练。
- `风险反馈`：用户可标注误报、漏报或确认风险，反馈进入本地 `risk_feedback`。

## 页面状态

| 状态 | 展示 |
|------|------|
| **加载中** | 报告骨架屏（6 个脉冲占位块） |
| **报告存在** | 完整报告渲染 |
| **报告不存在** | 404 页面："报告未找到" + 返回按钮 |
| **加载失败** | 错误提示 + 重试按钮 |

## API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/reports/[id]` | GET | 获取单份报告完整数据 |
| `/api/reports` | POST | 保存报告 |
| `/api/applications` | POST | 将报告加入本地投递清单 |
| `/api/risk-feedback` | GET/POST | 读取 / 提交风险识别反馈 |

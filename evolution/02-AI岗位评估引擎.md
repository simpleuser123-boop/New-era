# 02 — AI 岗位评估引擎

> 所属阶段：当前实现基线 · 核心模块

## 1. 评估流程总览

New Era 的 AI 岗位评估引擎采用 **单路径 SSE 流式架构**，当前产品不做 CLI 和 Web 双路径。评估结果可保存到 SQLite，并通过 `reportId` 进入报告、Tracker、面试教练和风险反馈链路。

```
JD URL/文本
    │
    ▼
┌─────────────────────┐
│ JD 内容获取           │
│  URL → Cheerio 抓取  │
│  文本 → 直接使用      │
└─────────┬───────────┘
          ▼
┌──────────────────────────────────────────────────────────┐
│ 第-1步: 假 AI 岗风险检测（并行）                            │
│                                                          │
│  Layer 1: 正则匹配 (20+ 条规则)                            │
│  Layer 2: 字典子串匹配 (AI 领域黑话词典)                    │
│  Layer 3: LLM 语义补充                                    │
│  → 输出: 风险等级 + 信号列表                                │
└─────────┬───────────────────────────────────────────────┘
          ▼
┌─────────────────────────────────────────────────────────────┐
│                    A-F 6 维评估（AI 专属）                   │
│                                                             │
│  A: 岗位定位 (15%)    AI 细分领域、公司 AI 战略位置           │
│  B: 技术栈匹配 (25%)  核心技术、框架、工具链逐条对照          │
│  C: 经验要求 (20%)    落地经验、开源贡献、论文/专利            │
│  D: 成长价值 (15%)    学习空间、技术前瞻性、团队氛围            │
│  E: 薪酬竞争力 (10%)  AI 行业基准薪资对比                    │
│  F: 风险合规 (15%)    假 AI 岗信号、合同陷阱、竞业限制          │
└─────────┬───────────────────────────────────────────────────┘
          ▼
┌─────────────────────┐
│ SSE 流式输出          │
│ A → B → C → D → E →F│
│ 每维推送后前端渲染     │
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│ 持久化到 SQLite       │
│ reports 表            │
└─────────────────────┘
```

当前实现补充（2026-06-11）：
- `/api/evaluate` 和 `/api/evaluate/stream` 均为 Node runtime。
- 报告保存后可通过 `/evaluate/report?reportId={id}` 读取。
- 报告页可加入本地投递清单、进入面试教练、提交风险反馈。
- 模型不可用时相关页面必须明确 fallback，不伪装成实时模型评估。

## 设计思想

AI 岗位评估引擎的维度设计遵循一个核心原则：**AI 从业者关心的不是"我适不适合"，而是"这个岗位在 AI 生态的什么位置"**。传统求职匹配算法回答"你的技能匹配这个岗位 85%"，但一个 LLM 应用工程师面对 AI Infra 岗位和算法研究岗位时，需要的是完全不同的判断框架。前者看的是工程落地能力，后者看的是论文产出和学术影响力。同一个人的同一份简历，在这两个方向上的竞争力完全不同。

引擎的 6 个维度彼此正交——技术栈匹配高不代表成长价值高（可能是用老旧技术栈维护遗留系统），薪酬竞争力高不代表岗位真实性高（可能是培训贷伪装）。每个维度独立评分，用户自主判断：薪酬低了但技术前瞻性强，是否还值得投？技术栈匹配但公司 AI 战略模糊，是否要去面试？

## 2. A-F 评分详解

### A 板块 — 岗位定位（15%）

提取 JD 的结构化信息，聚焦 AI 生态位：

- **AI 细分领域**：LLM 应用层 / AI Infra（中间件&工具链）/ 算法研究 / 具身智能 / AI 安全 / 其他
- **公司 AI 战略位置**：核心 AI 业务 / 赋能传统业务 / 探索性项目 / 跟风蹭热点
- **团队阶段**：成熟产品迭代 / 0-1 新项目 / 研究院/实验室
- **一句话 TL;DR**：这个岗位在 AI 生态中的角色

### B 板块 — 技术栈匹配（25%）

核心匹配逻辑 — 读取用户简历，逐条映射 JD 技术要求：

| JD 技术要求 | 简历匹配项 | 匹配度 | 说明 |
|------------|----------|--------|------|
| LangChain/LlamaIndex 开发经验 | 项目 #2 | 强 | 主导过 RAG 系统搭建 |
| 大模型微调 (LoRA) | — | 缺口 | 有 PyTorch 基础，可快速上手 |

**AI 技术栈分类映射**：
- LLM 应用层：LangChain / LlamaIndex / RAG / Agent / Prompt Engineering
- AI Infra：向量数据库 (Milvus/Pinecone) / 模型 serving / GPU 集群管理
- 算法研究：PyTorch / 模型训练 / 论文发表 / 数据集构建
- 具身智能：ROS / 机器人控制 / 仿真环境
- AI 安全：对抗样本 / 模型对齐 / 红队测试

### C 板块 — 经验要求（20%）

AI 从业者特有的经验维度：

| 经验维度 | 评估方式 |
|---------|---------|
| 落地经验 | 是否有从 0 到 1 的 AI 产品/系统交付 |
| 开源贡献 | GitHub stars / PR / 知名项目贡献 |
| 论文/专利 | 顶会论文 (NeurIPS/ICML/ACL...) 或相关专利 |
| 技术博客/分享 | 行业影响力 |

### D 板块 — 成长价值（15%）

评估这个岗位对 AI 从业者职业发展的长期价值：

- **技术前瞻性**：用的是主流技术栈还是前沿技术？
- **学习空间**：团队是否有资深 AI 工程师可以学习？
- **公司 AI 投入**：公司在 AI 方向是否有持续投入（融资/产品/团队扩张）？
- **赛道前景**：这个 AI 细分领域是否在上升期？

### E 板块 — 薪酬竞争力（10%）

基于 AI 行业基准数据：

| 层级 | 范围 | 来源 |
|------|------|------|
| LLM 应用层 (3-5 年) | 25K-45K/月 | 智联 2025 AI 人才报告 |
| AI Infra (5+ 年) | 35K-60K/月 | 脉脉人才迁徙报告 |
| 算法研究 (应届) | 20K-35K/月 | 公开数据 |
| 具身智能 (3-5 年) | 30K-50K/月 | 行业估算 |

### F 板块 — 风险合规（15%）

不参与加权评分，独立定性评估。三个等级：
- **高可信** — 真实 AI 岗位，多数信号正面
- **谨慎推进** — 混合信号，有可疑之处
- **疑似虚假** — 多个假 AI 岗指标触发

## 3. SSE 流式输出

```
POST /api/evaluate/stream
Body: { "jd_text": "...", "resume_text": "..." }

SSE 事件序列:
  ├─ { type: "phase", phase: "scanning_risks" }
  ├─ { type: "risks", risks: [...] }
  ├─ { type: "phase", phase: "evaluating", section: "A" }
  ├─ { type: "section", section: "A", data: {...} }
  ├─ { type: "section", section: "B", data: {...} }
  ├─ ...
  ├─ { type: "score", overall: 4.2 }
  └─ { type: "done" }
```

## 5. 分数解读

| 分数 | 建议 |
|------|------|
| 4.5+ | 强烈建议投递 — AI 生态位清晰，技术栈匹配，成长空间大 |
| 4.0-4.4 | 建议投递 — 真实 AI 岗位，部分维度可优化 |
| 3.5-3.9 | 可以投，注意短板 — 重点关注风险维度和成长价值 |
| 3.0-3.4 | 不太推荐 — 可能是边缘 AI 岗或存在较多风险信号 |
| <3.0 | 建议放弃 — 疑似假 AI 岗或严重不匹配 |

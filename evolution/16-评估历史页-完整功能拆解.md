## 功能拆解

### 1. 页面定位

评估历史页用于承接首页和 JD 评估页的“查看历史”入口，帮助用户回看过往岗位评估结果。

```text
首页最近评估
  └─ 查看全部 -> /reports

JD 评估页
  └─ 查看评估历史 -> /reports
```

### 2. 搜索与筛选

- 搜索字段匹配职位标题和公司名称。
- 状态筛选使用分段按钮样式。
- 搜索和筛选叠加生效。
- 无结果时展示空状态，而不是空白列表。

### 3. 历史记录卡片

每条历史记录建议展示：

| 字段 | 说明 |
|------|------|
| 公司标识 | 文字 Logo 或公司简称 |
| 岗位标题 | 如 AI 产品经理（大模型） |
| 公司名称 | 如字节跳动 |
| 日期 | 最近评估日期 |
| 状态 | 已完成 / 评估中 / 需优化 |
| 评分 | 完成态显示综合评分 |
| 进度 | 评估中显示进度 |

### 4. 状态定义

| 状态 | 含义 | 展示 |
|------|------|------|
| `completed` | 报告已生成 | 评分 + 查看报告 |
| `processing` | SSE 或同步评估进行中 | 进度条 |
| `needsOptimization` | 分数较低或技能缺口明显 | 优化提示 |

### 5. 空状态

当搜索或筛选无匹配记录时展示：

- 空状态图标
- “没有匹配的评估记录”
- 操作建议
- 重置筛选按钮

## 数据来源

当前前端读取：

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/reports` | GET | 读取评估历史列表 |
| `/api/reports/[id]` | GET | 读取单份报告详情 |
| `/api/reports/[id]` | DELETE | 删除本地报告并解除关联引用 |

当前 `/api/reports` 支持 `limit`；搜索和状态筛选在前端列表中完成。

```text
GET /api/reports?limit=100
```

## 页面状态

| 状态 | 展示 |
|------|------|
| 加载中 | 列表骨架屏 |
| 有数据 | 筛选器 + 历史卡片列表 |
| 无匹配 | 空状态 + 重置筛选 |
| 加载失败 | 错误提示 + 重试 |
| 空历史 | 引导用户创建第一次评估 |
| 删除确认 | 展示对象、影响范围和不可恢复提示 |
| 删除失败 | 保留列表项并展示错误 |

## 验收标准

- `/reports` 可从侧边栏和评估页进入，当前导航高亮正确。
- 搜索关键词能即时过滤列表。
- 状态筛选与搜索能叠加使用。
- 空状态可一键重置筛选。
- 删除报告前必须出现确认弹窗，删除后历史列表移除该报告。
- 删除报告不物理删除关联投递、面试和简历优化记录。
